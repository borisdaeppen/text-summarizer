Why don’t my funnel pages show any visits?

There are a couple of reasons why your pages might not show any visits. The simplest reason is that you’re looking at the wrong filter settings. To see if this is the problem, simply set all of the filters to show “All”:



If your missing stats suddenly show up, then either your filter settings were wrong, or your visits are being attributed to the wrong category.

Of the four types of filters, the only one that generally confuses people is the difference between a “qualified” and an “unqualified” visitor. You can read about the difference here:
 
   What’s the difference between Qualified & Unqualified visitors?

If you’ve check your filter settings and your missing stats are still missing, then read on …


Your Page Settings URL is Incorrect

The other very common reason for page visits not showing up is that you’ve entered the wrong URL for the “Page URL” setting of your FunnelMagick page:
 


 

For FunnelMagick to work correctly, the URL you enter as the Page URL must be the actual, final page that the user sees. The setting cannot be a link that redirects to your final page, such as a link shortener like Bitly or TinyURL.

ClickFunnels is a popular Page Builder that uses a redirect link to the pages that you create with it. Because ClickFunnels is so popular, let’s use it as an example for how to set the proper Page URL in FunnelMagick even though ClickFunnels gives you a redirect link which won’t work.

Take a look at this opt-in page we created inside the ClickFunnels Page Builder:


 

Notice the generic ClickFunnels URL that it shows:

http://clickmagick.clickfunnels.com/optinog321qfo

That link will certainly display the opt-in page, but when you use it, look at the final URL in the Address Bar:
 




The final URL uses the antarcticsnowwater.com custom domain that we set up with ClickFunnels and uses the “path” optin-bottles that ClickFunnels asked us for when we created the page. That’s normal user behavior with ClickFunnels which is why so many ClickFunnel users have trouble with the Page URL setting in FunnelMagick. 

The way ClickFunnels works—and the way a LOT of Page Builders work—is that it gives you a link into its own system that then redirects to the final page that you created.

There’s absolutely nothing wrong with that approach, but for your visits to show up in your FunnelMagick stats, you have to use the final URL—not the initial URL—for the page.

Here’s why …

When a visitor lands on your web page, the piece of FunnelMagick tracking code that you inserted on the page is executed. The first thing that piece of code does is grab the URL of the page its on.

Next, the code looks at the Page URL setting of every page in your funnel to see which page just got visited. If you’ve set the Page URL to the initial page URL that ClickFunnels gives you, the funnel tracking code won’t find the page and no visit will be recorded!

This is why you must set the Page URL setting to the actual, final URL that the tracking code is going to execute from. If you don’t, your visits won’t be recorded.


So how do you ensure that you have the correct URL?

It’s pretty easy …

Just enter the URL that your Page Builder gives you and use the final URL that shows up in the Address Bar.

In this ClickFunnels example, we would enter ...

http://clickmagick.clickfunnels.com/optinog321qfo

… into the browser and once the has page loaded, the Address Bar would show:

http://antarcticsnowwater.com/optin-bottles/

This is the URL we have to use in the Page URL setting with FunnelMagick.

If you enter a URL into your browser and the same URL is still showing after the page has been displayed, that simply means that your Page Builder isn’t using any redirected links. Yay!


Correcting the Page URL Setting

If you discover that you’ve entered the wrong URL into your Page URL setting, all you need to do is enter the correct URL.

There is one hitch …

If you’ve created any Entry Links to the funnel, all of those entry links will point to the old incorrect Page URL. You’ll need to delete those entry links and recreate them after you’ve updated the Page URL.

In fact, FunnelMagick won’t even let you change the Page URL until you’ve deleted the erroneous Entry Links. So, delete the Entry links, change the Page URL, then recreate your Entry Links and you should be good to go.
