Why isn’t my content mobile responsive?

ClickMagick itself is “mobile friendly” so if you’re having problems with your MagickPops, MagickBars and pages not workign well with mobile devices, you need to make sure that the content you create is mobile responsive.

With that being said, if you’re using Custom Domain Method #1 (which is no longer recommended), it’s quite possible that your domain registrar is forwarding or cloaking your domain in a way that doesn’t allow mobile responsiveness. This is something we have no control over, and every domain registrar and hosting company are different in terms of how they handle domain forwarding …

So, if you’re using Custom Domain Method #1 and you’re positive that your content is mobile responsive but it’s still not displaying properly on mobile devices, you have a couple of options:
 
1.    You can turn off cloaking at your domain registrar and just redirect instead. You can then use ClickMagick’s built-in cloaking using our www.clkmg.com tracking domain

or …
 
2.    You can switch to using a CNAME DNS custom domain, or to Script Forwarding. Both approaches work fine with mobile responsive websites, but we recommend using a CNAME DNS custom domain since it is the most flexible setup and easier to set up.
